# Repository Instructions

## Project Progress Update Rule

For every material task in this repository, maintain:
C:\claudcode_ap\growth_lab_core\Project_Progress.md

Required behavior:
1. At task start, update Current Status and In Progress only when the status materially changes.
2. At task completion, update Completed, Next Actions, Issues and Risks, and Recent Updates.
3. Use one unique Work ID for each task and do not create duplicate history rows.
4. Update CHANGELOG.md only for design, specification, implementation, configuration, dependency, or operational rule changes.
5. Do not write credential values, Recovery codes, Passwords, Tokens, Secrets, API Keys, or Authenticator seeds.
6. Do not read or expose credential file contents. If a possible credential file is detected, report only the minimum information required for Human Owner action.
7. Do not mark a security-sensitive item complete unless verification evidence exists.
8. Preserve existing project history and do not rewrite unrelated progress records.
